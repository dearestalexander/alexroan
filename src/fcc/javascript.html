Document object model (DOM)

A web application programming interface (API) used to build websites

  • Actually consists of multiple individual APIs that work together
  • Represents web pages in memory so that programs can change the document structure, style, and content
    • This is an ‘object orientated’ representation of the web page
  • This representation of the web page can be modified by a scripting language such as JavaScript

Structured as a logical tree where:

  • Each branch of the tree ends in a node
  • Each node contains objects: which can represent properties, methods, events etc.
  • It’s common to refer to nodes in the DOM as elements, but not every node is an element

API

An API is a way for two or more programs to communicate. The DOM includes multiple APIs, for example:

  • Core DOM: entities describing any document and objects within
  • HTML DOM API: support for representing HTML documents to the core DOM
  • SVG API: support for representing SVG documents

Interfaces

The DOM is made up of interfaces. Each of these interfaces has different methods. For example the interfaces include:

  • Document
  • DocumentType
  • DocumentFragment
  • Element
  • CharacterData
  • Attr

Some examples of common interfaces and methods are:

  • Document interface, querySelector method
    • const constName = document.body.querySelector(".className");

    • setText.addEventListener("click", () => {}
  • Node interface, textContent method:
    • constName.textContent = "Sample text to update";

The relationship between objects and interfaces that are implemented in the DOM can be confusing.

  • The form element gets name from HTMLFormElement interface, but class from HTMLElement interface
  • Many objects implement several different interfaces

Objects

Object basics

Real life objects:

  • Real life object: car.name = fiat, car.model = 500, car.weight = 850kg, car.color=white
  • Real life car has methods like; car.start(), car.drive(), car.brake(), car.stop()
  • Car objects have the same properties, but values differ from car to car
  • Car objects have the same methods, but methods are performed at different times

JavaScript objects:

  • { key: value }
  • Non-primitive data types
  • Contain properties in the form of key:value pairs (sometimes referred to as name:value)
  • Similar to arrays, except instead of indexes, use properties to access data
  • const car = {type: Fiat, model: "500", color: "white"}; // common practice to use const

Key points:

  • Objects are containers for Properties and Methods
  • Methods are Functions stored as Properties
  • Properties can be primitive values, functions, or even other objects
  • (All JavaScript values, except primitives, are objects).

Object destructuring syntax

  • As an alternative to dot notation
  • Allows you to unpack values from arrays and objects.
  • let { property1: variable1, property2: variable2 } = object;

Variables

Variable definition

  • A variable points to a specific memory address
  • Use let for when you may reassign the value
  • Use const when you do not plan to reassign values
    • const will tell JavaScript to throw an error if you accidentally re-assign it
    • A const variable cannot be uninitialised i.e. const exVariable;
  • Use camelCase for variable names
    • The first word in lowercase
    • The first letter of every following world capitalised
  • A variable declared without a value will have the value undefined
  • Variable names can include letters, numbers, dollar signs, and underscores
  • Variable names cannot contain spaces and must not begin with a number
  • = is used as the assignment operator
  • Initialization is the assignment of a value at declaration
  • You can assign the value of a variable to another variable.

Guidelines:

  1. Always declare variables
  2. Always use const if the value should not be changed
  3. Always use const if the type should not be changed (Arrays and Objects)
  4. Only use let if you can’t use const
  5. Only use var if you MUST support old browsers.

Variable scope

  • Where a variable is declared determines where it can be used

  • Next is block scope or local scope:
    • It applies to those declared inside a block of code
    • They are available only within that block
    • If you try to access them outside their scope you get a reference error

const insertionSort = (array) => {
  for (let i = 1; i < array.length; i++) {
    const currValue = array[i];
    let j = i - 1;

      array[j + 1] = array[j];

    }

  }

  return " ".repeat(rowCount - rowNumber) + character.repeat(2 * rowNumber - 1) + " ".repeat(rowCount - rowNumber);

Using an object to count occurrence

  • Compare each array element with an object:

    • If it doesn’t exist set equal to one

  if (counts[el]) {
    counts[el] += 1;
  } else {
    counts[el] = 1;
  }
})
if (new Set(Object.values(counts)).size === 1) {
  return null;
};

Theory

Call stack

In computer science, a stack is a data structure where items are stored in a LIFO (last-in-first-out) manner. If you imagine a stack of books, the last book you add to the stack is the first book you can take off the stack. Or an array where you can only .push() and .pop() elements.

The call stack is a collection of function calls stored in a stack structure. When you call a function, it is added to the top of the stack, and when it returns, it is removed from the top / end of the stack.

Recursion

A recursive function is a function that calls itself over and over.

  • Take care to avoid infinite loop
  • Use a base case to stop the function from calling itself, good practice to write this first.
  • Use the recursive case to call itself
  • What is the least amount of work to get closer to the base case

Const vs. let

  • With variables the data is created first, then the variable label is pointed to it
  • Re-assignment
    • is pointing the variable label to a new value
    • It isn’t modifying the data, it detaches from existing data and attaches to new data
  • Let vs. const
    • With const we create an indestructible link between the variable name and data, but we can still modify the data.This is known as mutation
      • Adding an entry to an array
      • Editing the key: value pairs in an object

Primitive data types like strings and numbers are immutable

  • In the below example we aren’t editing 36 to become 37, we are re-assigning the variable to a different number
leg age = 36;
age = 37;

  • Can be used on objects and arrays to prevent all form of mutations
  • It’s shallow, it won’t freeze nested objects
